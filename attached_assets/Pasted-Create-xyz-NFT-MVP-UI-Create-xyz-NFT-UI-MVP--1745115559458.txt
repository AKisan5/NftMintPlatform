Create.xyzでNFT発行アプリのMVP UIを再現するための指示
以下は、Create.xyzでNFT発行アプリのUI MVPを再現するための指示です。この指示に従って、イベント情報管理画面とNFTミント画面のUIのみを構築してください。
## 重要な注意点
これはUIの再現のみが目的です。実際の機能実装（データベース連携、ブロックチェーン接続、実際のNFTミントなど）は現段階では不要です。見た目と画面遷移のみを再現してください。
## 全体のデザインテーマ
- カラースキーム: 青系統（メインカラー: #3B82F6、アクセント: #2563EB）
- 背景: 薄い青のグラデーション（#EFF6FF から白へ）
- フォント: システムデフォルトフォント
- 角丸デザイン: すべてのカード、ボタン、入力フィールドに適用
- レスポンシブデザイン: モバイルとデスクトップの両方に対応

## 1. イベント情報管理画面
### 画面構成
1. ヘッダー部分
1. タイトル: 「参加証NFT発行アプリ」（大きめのフォント、青色）
2. サブタイトル: 「イベント管理画面」（中サイズフォント、薄い青色）

2. イベント情報入力フォーム
1. 白背景のカード形式（影付き）
2. カードヘッダー: 薄い青背景、「イベント情報登録」というタイトル
3. 以下の入力フィールドを縦に配置:
1. イベント名（テキスト入力）
2. イベント詳細（テキストエリア）
3. ミント開始日時（日付選択 + 時間選択）
4. ミント終了日時（日付選択 + 時間選択）
5. NFTの発行上限数（数値入力）
6. ガス代肩代わり設定（ラジオボタン: 「肩代わりしない」または「肩代わりする」）
7. NFTの譲渡設定（ラジオボタン: 「譲渡可」または「譲渡不可」、説明文付き）
8. 合言葉（テキスト入力、鍵アイコン付き）



3. 配布するNFTセクション
1. 内部カード形式（薄い青のヘッダー付き）
2. 「配布するNFT」というタイトル（画像アイコン付き）
3. 左側にNFT画像アップロード/プレビューエリア
1. 画像がない場合は「NFT画像をアップロード」「クリックして選択」と表示
2. 画像がある場合はプレビュー表示、ホバーで「画像を変更」オーバーレイ表示

4. 右側にNFT情報入力フィールド
1. NFT名（テキスト入力）
2. NFT説明（テキストエリア）
3. 動画アニメーション（任意、ファイル選択）

5. 「このNFTは一般参加者に配布されます」という説明文

4. 送信ボタン
1. 「イベント情報を登録」（青色、幅いっぱい）

5. 登録成功時の表示
1. 成功メッセージ（緑色の通知）
2. イベント情報の表示（青色の背景パネル）
1. イベント名、期間、合言葉、NFT名、ガス代肩代わり設定、譲渡設定

3. イベント参加者用URLの表示（コピーボタン付き）
4. 「イベント参加者用QRコードにする」ボタン
5. 「新しいイベントを登録」ボタン

6. QRコードダイアログ
1. ダイアログタイトル: 「イベント参加者用QRコード」
2. QRコード表示エリア（ダミーQRコード）
3. イベント名の表示
4. 「閉じる」と「QRコードを保存」ボタン


## 2. イベント参加者画面
### 画面構成
1. ヘッダー部分
1. タイトル: 「参加証NFT発行アプリ」（大きめのフォント、青色）
2. サブタイトル: 「イベント参加者画面」（中サイズフォント、薄い青色）

2. メインカード
1. 白背景のカード形式（影付き）
2. カードヘッダー: 薄い青背景、状態に応じたタイトル
1. 「合言葉の入力」または「ウォレット接続」または「NFTミント」

3. イベント情報表示エリア（青色の背景パネル）
1. イベント名、詳細、ミント期間の表示



3. ステップ1: 合言葉入力フォーム
1. 合言葉入力フィールド（鍵アイコン付き）
2. 「合言葉を確認」ボタン（青色、幅いっぱい）
3. エラー時のメッセージ表示（赤色）
4. 成功時のメッセージ表示（緑色）

4. ステップ2: ウォレット接続
1. 説明テキスト（青色の通知）
2. 「ウォレットを接続する」ボタン（青色、ウォレットアイコン付き）
3. 接続中の表示（ローディングアイコン）

5. ステップ3: NFTミント
1. NFT画像のプレビュー表示（中央配置）
2. 「NFTをミントする」ボタン（青色、幅いっぱい）
3. ミント中の表示（ローディングアイコン）
4. ミント成功時の表示（緑色のチェックマークと成功メッセージ）


## 画面遷移の設定
1. イベント管理画面
1. 「イベント情報を登録」ボタンクリック → 登録成功表示に切り替え
2. 「イベント参加者用QRコードにする」ボタンクリック → QRコードダイアログ表示
3. 「新しいイベントを登録」ボタンクリック → フォームをリセット

2. イベント参加者画面
1. 「合言葉を確認」ボタンクリック → 正しければウォレット接続ステップへ、間違っていればエラー表示
2. 「ウォレットを接続する」ボタンクリック → 接続中表示 → NFTミントステップへ
3. 「NFTをミントする」ボタンクリック → ミント中表示 → ミント成功表示

## 実装上の注意点
1. 機能実装は不要
1. ボタンクリックで画面遷移するだけで、実際のデータ保存や検証ロジックは実装不要
2. ローカルストレージやデータベースの実装は不要
3. 実際のブロックチェーン接続やNFTミント機能は実装不要

2. 見た目の再現に集中
1. レイアウト、色、アイコン、フォームの配置を忠実に再現
2. レスポンシブデザインを意識（モバイルでも使いやすく）
3. 特に「配布するNFT」セクションの画像アップロード/プレビュー機能の見た目を重視

3. ダミーのインタラクション
1. 合言葉の検証: 「aikotoba」と入力された場合のみ次のステップに進む
2. ウォレット接続: ボタンクリック後、短い遅延の後に接続完了とする
3. NFTミント: ボタンクリック後、短い遅延の後にミント成功とする
4. 画像アップロード: 画像選択後、プレビュー表示に切り替わる

4. NFT画像選択UI
1. 「配布するNFT」カード内の左側に画像アップロード/プレビューエリアを配置
2. クリックで画像選択ダイアログが開く
3. 画像選択後はプレビューが表示され、ホバーすると「画像を変更」オーバーレイが表示される
4. モバイル表示では画像エリアが上部に、情報入力が下部に配置される

このMVPが完成したら、次のステップとして実際の機能実装（データベース連携、実際のNFTミント機能など）を検討します。まずはUIの使い勝手と見た目を確認することが目的です。
